# Indian Sign Language Recognition (ISL)

Indian Sign Language Recognition (ISL) is a capstone project that combines a Vue frontend, a Flask backend, and a YOLO-based image recognition model to identify Indian Sign Language gestures from uploaded images.

## Features

- Upload an image containing an ISL hand gesture
- Run gesture recognition through the Flask API
- View the predicted label, confidence score, and annotated image
- Browse a polished multi-page interface for the project overview, team, and registration flow

## Tech Stack

- Frontend: Vue CLI, Vue Router, Bootstrap
- Backend: Flask, Flask-CORS
- ML: Ultralytics YOLO
- Python environment: local `.venv`

## Setup

### 1. Clone the project

```bash
git clone <your-repo-url>
cd "Indian Sign Language Recognition(ISL)"
```

### 2. Create and activate the Python virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Install backend dependencies

```bash
pip install -r backend/requirements.txt
```

### 4. Install frontend dependencies

```bash
cd frontend
npm install
cd ..
```

## Running the Project

### Start the backend

```bash
source .venv/bin/activate
python backend/main.py
```

The backend runs on:

```text
http://127.0.0.1:5001
```

### Start the frontend

Open a second terminal:

```bash
cd frontend
npm run serve
```

The frontend runs on a Vue dev server URL similar to:

```text
http://127.0.0.1:8080
```

If port `8080` is already in use, Vue may automatically move to another port such as `8081`.

## API Endpoints

### `GET /`

Returns backend status and model readiness.

### `POST /upload`

Accepts a multipart form upload with the field name `file`.

Response includes:

- `class_label`
- `confidence`
- `total_spots`
- `annotated_image`

### `GET /result`

Returns the latest annotated result image generated by the backend.

## Notes

- The backend is configured to run on port `5001` because port `5000` may already be in use on some macOS systems.
- The latest result image is saved to `backend/images/result.jpg`.
- The register page currently stores data locally in the UI only. It can be connected to a database or email workflow later.
